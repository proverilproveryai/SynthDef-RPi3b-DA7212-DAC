## Концепция

UI — это **рантайм-движок**, который рендерит контролы по декларативному описанию (`ui_schema.json`), а не хардкодом. Это позволяет:
- переключать пресеты с разными наборами контролов без перезагрузки страницы
- в будущем генерировать интерфейс через LLM — модель генерирует схему, не произвольный HTML

Визуальный стиль — тёмная тема, монохром + один акцентный цвет (`#e8ff3a`), шрифт JetBrains Mono. Референс: edcreativecoder.com.

## Навигация

Бургер-меню в правом верхнем углу шапки. Три экрана:

```
┌─────────────────────────────────────────────┐
│  PIPEDALUS              [status] [≡]         │  ← шапка (fixed)
│─────────────────────────────────────────────│
│                                              │
│  <текущий экран>                             │
│                                              │
│─────────────────────────────────────────────│
│  ▐▌ −12dB  [MASTER ────────────●──] 0.8    │  ← пиковый индикатор + master (fixed)
└─────────────────────────────────────────────┘
```

При нажатии бургера — оверлей с меню съезжает справа:
```
MENU
────────────────
PRESETS
EDITOR
SETTINGS
────────────────
v0.1.0
```

## Экраны

### 1. PRESETS

Стартовый экран. Список карточек сохранённых пресетов + кнопка создания нового.

```
MY PRESETS

┌─────────────────────────────────────────┐
│  murf_default              ›            │
│  8-band filter array                    │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│  +  NEW PRESET                          │
└─────────────────────────────────────────┘
```

- Тап на карточку → загружает пресет (WS: `preset_load`) и переходит на EDITOR
- Кнопка `+ NEW PRESET` → диалог создания (в будущем — запрос к LLM)

### 2. EDITOR

Экран редактирования/управления текущим пресетом. Два режима:

**PLAY MODE** (дефолт) — контролы активны, изменения летят в OSC. Никаких UI-инструментов редактирования.

**EDIT MODE** (кнопка `EDIT` в тулбаре) — поверх контролов отображается сетка, виджеты можно добавлять, перемещать, удалять. Сохранение записывает `ui_schema.json` на Pi.

```
murf_default                        [EDIT]

┌── canvas ───────────────────────────────┐
│                                         │
│   [виджеты из ui_schema.json]           │
│                                         │
└─────────────────────────────────────────┘
```

### 3. SETTINGS

Глобальные настройки системы.

```
WI-FI
┌─────────────────────────────────────────┐
│  Mode    HOTSPOT (PiPedalus-EF66)       │
│  [ SWITCH TO WI-FI ]                   │
└─────────────────────────────────────────┘

AUDIO
┌─────────────────────────────────────────┐
│  Device  hw:0 (DA7212)                 │
│  Rate    48000 Hz                       │
│  Buffer  256                            │
│  [ REFRESH DEVICES ]                   │
└─────────────────────────────────────────┘

SYSTEM
┌─────────────────────────────────────────┐
│  CPU     23%                            │
│  RAM     412 MB                         │
│  scsynth running                        │
└─────────────────────────────────────────┘
```

## Preset Editor — подробно

### Сетка

Холст редактора разделён на ячейки. Количество колонок фиксировано (8 для MuRF), но физический размер ячейки вычисляется в рантайме из ширины canvas (см. [Адаптив сетки](#адаптив-сетки)). В `ui_schema.json` позиция хранится в единицах колонок — схема не зависит от устройства.

```
col →  0    1    2    3    4    5    6    7
row ↓  ┼────┼────┼────┼────┼────┼────┼────┼
  0    │         knob(2×2)    │  toggle    │
  2    │ fader  │ fader  │ fader  │ fader  │
  4    │    step-grid (8×2)               │
  6    └──────────────────────────────────┘
```

### PLAY MODE vs EDIT MODE

| Действие | PLAY | EDIT |
|---|---|---|
| drag по виджету | изменяет значение параметра | перемещает виджет |
| tap | активирует/тоглит | выделяет виджет |
| long press | — | показывает меню (удалить, настроить) |
| `+` кнопка | нет | добавить виджет из библиотеки |
| `DONE` | — | сохраняет схему на Pi |

### Добавление виджета (EDIT MODE)

Нажатие `+` открывает bottom sheet с библиотекой виджетов:

```
ADD WIDGET
──────────────────────
◎  KNOB        →
▬  FADER       →
■  STEP GRID   →
⠿  PIANO ROLL  →
▪  TOGGLE      →
▾  SELECT      →
```

### Настройка виджета

Long press в EDIT MODE открывает панель настройки:

```
KNOB SETTINGS
──────────────────────
Label   RATE
OSC     /murf/param/rate
Min     0.1
Max     10.0
Default 2.0
──────────────────────
[ DELETE ]  [ DONE ]
```

## Адаптив сетки

Ширина ячейки вычисляется динамически из фактической ширины canvas:

```js
cell_size = Math.floor((canvas_width - padding) / grid_cols)
```

При ресайзе `ResizeObserver` на canvas пересчитывает `cell_size` и все виджеты переезжают автоматически. В `ui_schema.json` позиции хранятся только в единицах колонок (целые числа), пиксели — только в рантайме.

Примерные значения `cell_size` на разных устройствах при 8 колонках и 16px паддинга с каждой стороны:

| Устройство | viewport | cell_size |
|---|---|---|
| iPhone SE | 375px | 42px |
| iPhone 14 | 390px | 44px |
| iPhone 14 Plus | 430px | 49px |
| iPad mini | 744px | 88px |

На планшете виджеты просто становятся крупнее — сетка та же, физические размеры больше. Это желательное поведение для живого выступления.

## UI Kit — Web Components

Каждый виджет — нативный Web Component (`customElements.define`). Без сборки/бандлинга, с инкапсуляцией через Shadow DOM.

### Контракт виджета

**Атрибуты:** `id`, `label`, `osc`, `param-id`, `min`, `max`, `value`

**Исходящее событие:** диспатчит `ui-change` при изменении пользователем:
```js
detail: { paramId: 'rate', osc: '/murf/param/rate', value: 2.0 }
```

**Входящий метод:** `widget.update(value)` — для виджетов, получающих данные от сервера (метр, спектр). `preset-renderer.js` вызывает этот метод при входящих WS-сообщениях.

`preset-renderer.js` слушает `ui-change` на контейнере и транслирует в WebSocket:
```js
PiPedalus.send({ type: 'osc', address: e.detail.osc, value: e.detail.value })
```

Node.js получает это и шлёт OSC напрямую по адресу — **сервер не знает ничего про конкретные эффекты**, он просто транслятор. Добавление нового эффекта с новыми OSC-адресами не требует правки сервера.

### Файловая структура

```
web/public/
├── index.html
├── style.css
├── app.js                  ← WS-клиент + роутер + глобальный API
├── preset-renderer.js      ← читает ui_schema, монтирует виджеты
├── components/
│   ├── ui-knob.js
│   ├── ui-fader.js
│   ├── ui-toggle.js
│   ├── ui-step-grid.js
│   ├── ui-piano-roll.js
│   ├── ui-select.js
│   └── ui-meter.js         ← пиковый индикатор (read-only, данные от сервера)
└── screens/

    ├── preset-editor.js
    └── settings.js
```

## Виджеты

### ui-knob
Вращательный регулятор. Управление: drag вверх/вниз по вертикали (не вращение — на мобилке удобнее).

### ui-fader
Вертикальный или горизонтальный фейдер. Ориентация задаётся атрибутом `orient="vertical|horizontal"`.

### ui-toggle
Переключатель on/off. Диспатчит `0` или `1`.

### ui-step-grid
Паттерн-сетка N×M. Тап по ячейке тоглит её состояние. Диспатчит весь паттерн как плоский массив `[0,1,0,1,...]`.

### ui-piano-roll
2D степ-грид где X = время (шаги), Y = высота ноты. Поддерживает pinch-zoom по оси Y для работы с узким диапазоном нот на маленьком экране. Самый сложный виджет — реализуется после базового набора.

Атрибуты: `steps`, `note-min`, `note-max`, `param-id`, `osc`.

### ui-select
Выпадающий список или сегментированный контрол (если вариантов ≤ 4).

### ui-meter *(read-only)*
Пиковый индикатор уровня сигнала. Не генерирует события — только принимает `update(db)`. Реализован как часть фиксированной нижней панели (не добавляется в схему пресета).

## UI Schema — формат

Каждый пресет содержит `ui_schema.json`:

```json
{
  "grid": { "cols": 8, "cell": 40 },
  "widgets": [
    {
      "id": "w_rate",
      "type": "knob",
      "col": 0, "row": 0, "w": 2, "h": 2,
      "label": "RATE",
      "paramId": "rate",
      "osc": "/murf/param/rate",
      "min": 0.1, "max": 10, "value": 2.0
    },
    {
      "id": "w_drive",
      "type": "knob",
      "col": 2, "row": 0, "w": 2, "h": 2,
      "label": "DRIVE",
      "paramId": "drive",
      "osc": "/murf/param/drive",
      "min": 0, "max": 1, "value": 0.3
    },
    {
      "id": "w_lfo",
      "type": "toggle",
      "col": 4, "row": 0, "w": 2, "h": 1,
      "label": "LFO",
      "paramId": "lfo_on",
      "osc": "/murf/param/lfo_on",
      "value": 0
    },
    {
      "id": "w_pattern",
      "type": "step-grid",
      "col": 0, "row": 2, "w": 8, "h": 2,
      "label": "PATTERN",
      "paramId": "pattern",
      "osc": "/murf/pattern/load",
      "rows": 8, "steps": 8
    }
  ]
}
```

`osc` — точный адрес, который уйдёт в sclang. `paramId` — внутренний идентификатор для логов и состояния. Схема самодостаточна: `preset-renderer.js` и Node.js не содержат никаких маппингов параметров.

## Пиковый индикатор и Master Volume

Фиксированная панель внизу экрана (над debug-логом), присутствует на всех экранах:

```
┌─────────────────────────────────────────────┐
│  ▐▌ −12dB   MASTER [────────────●──]  0.8  │
└─────────────────────────────────────────────┘
```

**Пиковый индикатор (`<ui-meter>`):**
- sclang считает амплитуду через `Peak.kr` на мастер-бусе
- шлёт OSC → Node.js с частотой ~25 раз/сек
- Node.js транслирует в WS: `{ type: 'meter', db: -12.3 }`
- `ui-meter.update(-12.3)` обновляет отображение
- нагрузка минимальна: 25 небольших пакетов/сек

**Master Volume:**
- обычный горизонтальный фейдер, всегда видимый
- OSC: `/master/volume`
- не является частью `ui_schema.json` пресета — это глобальный параметр

## Роутер

Простой hash-роутер без библиотек:

```js
window.Router.navigate('preset-editor', { preset: 'murf_default' });
window.Router.navigate('settings');
```

При навигации:
1. рендерит HTML нужного экрана в `#app`
2. вызывает `screen.mount(container, params)`
3. обновляет активную ссылку в меню

## WebSocket-команды UI

| Направление | Тип | Описание |
|---|---|---|
| → сервер | `preset_load` | загрузить пресет по имени |
| → сервер | `schema_save` | сохранить ui_schema (из EDIT MODE) |
| → сервер | `osc` | произвольная OSC-команда `{address, value}` |
| → сервер | `audio_device_list` | запросить список аудиоустройств |
| → сервер | `wifi_connect` | переключить в STA-режим |
| → сервер | `wifi_switch_to_ap` | переключить в AP-режим |
| ← сервер | `status` | текущий статус при подключении |
| ← сервер | `meter` | пиковый уровень `{db: -12.3}` (~25/сек) |
| ← сервер | `audio_device_list` | список устройств ALSA |
| ← сервер | `wifi_switching` | предупреждение перед сменой сети |
| ← сервер | `wifi_ok` | сеть переключена успешно |
| ← сервер | `wifi_error` | ошибка переключения сети |
